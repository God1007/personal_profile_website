# Technical Brand Redesign

**Date:** 2026-04-16

## Goal

Redesign the current personal portfolio and blog site into a more expressive technical brand website. The result should feel more alive, more intentional, and more aligned with a developer identity, while remaining polished, readable, and deployment-friendly.

## Approved Direction

The redesign should follow these confirmed decisions:

- overall style: premium technical brand
- not hacker-terminal style
- not flat portfolio-template style
- supports both light mode and dark mode
- light mode: elegant technical brand
- dark mode: stronger engineering atmosphere
- same layout across both themes
- default theme follows system preference
- manual theme toggle is available
- motion intensity: medium

## Core Visual Intent

The current site feels too linear, too uniform, and too visually flat. The redesign should introduce:

- stronger rhythm in layout
- more hierarchy between sections
- more expressive typography
- layered backgrounds
- a clearer technical identity
- restrained but visible motion

The site should feel like the homepage of a technically strong individual with taste, not just a collection of cards stacked down the page.

## Visual Language

### Shared brand language

Both themes should share:

- editorial-style typography hierarchy
- strong English title moments
- Chinese explanatory copy
- spacious section transitions
- layered cards and surfaces
- subtle technical motifs such as grid lines, glow halos, separators, and interface framing

### Light theme

The light theme should feel:

- bright
- crisp
- premium
- structured

Recommended characteristics:

- off-white or near-white base
- cool grays
- blue-cyan accent highlights
- soft gradients and subtle grid overlays
- quiet shadows

### Dark theme

The dark theme should feel:

- deeper
- more technical
- more atmospheric
- more engineering-oriented

Recommended characteristics:

- dark charcoal or blue-black base
- high-contrast text
- cool electric accent tones
- stronger glow treatment
- more pronounced layering

The dark theme should still feel mature and controlled, not flashy or gamer-like.

## Homepage Structure

The homepage should be restructured into these major sections:

1. Hero
2. Profile Snapshot
3. Selected Projects
4. Technical Notes
5. Education & Journey
6. Contact
7. Footer

## Section Design

### Hero

The hero should become the main brand statement.

Layout direction:

- left side: a large English headline
- supporting Chinese description beneath or nearby
- right side: compact profile facts or an identity panel
- CTA buttons for projects and writing

The hero should also include visual atmosphere:

- grid or line overlays
- subtle glow or gradient field
- layered decorative framing

Possible English title style:

- `Engineering Systems with Clarity`
- `Networks, Code, and Practical Intelligence`
- `From Protocols to Real Systems`

The final implementation can pick one clear headline and keep the rest of the copy in Chinese.

### Profile Snapshot

This section should replace the current plain descriptive block with a more branded identity layer.

It should present:

- name
- current degree / institution
- technical focus
- language / skills highlights
- quick credibility points

This should feel more like a composed profile panel or technical identity block than a generic “about me” paragraph.

### Selected Projects

This section should no longer feel like three equal cards in a generic grid.

It should present projects with stronger hierarchy:

- one larger featured project card
- secondary cards for additional items
- richer metadata treatment
- more system-case-study feeling

The section should communicate:

- what the project is
- why it matters
- what technologies were involved
- what capability it proves

### Technical Notes

This section replaces the softer “Writing & notes” feel with a stronger developer-facing writing identity.

Recommended title style:

- `Technical Notes`

Chinese explanation can sit beneath it.

This section should visually feel closer to a technical reading archive:

- article cards with metadata
- slightly more editorial structure
- stronger entry points into the blog

### Education & Journey

This section should remain but feel more timeline-driven and less list-like.

It should visually communicate progression:

- undergraduate foundation
- postgraduate progression
- research contribution

### Contact

The contact area should feel more direct and purposeful.

It should feel like an invitation to connect on technical work, collaboration, or opportunity, rather than a generic website footer CTA.

## Motion and Interaction

Motion intensity is approved at medium.

### Included motion

- entrance fade and slight upward reveal for hero and sections
- theme-switch transition
- subtle background halo movement
- hover lift and depth changes on cards
- scroll-based reveal for sections
- gentle emphasis on buttons and interactive elements

### Excluded motion

- heavy particles
- complex 3D
- large canvas animations
- distracting or constant movement

The motion should add energy and polish without reducing readability or performance.

## Theme System

The redesign should include a real theme system rather than a color swap.

### Behavior

- default follows system preference
- user can manually switch theme
- preference is persisted locally

### Scope

The theme system should affect:

- background
- surface colors
- text colors
- borders
- glows
- shadows
- accents
- technical decorative layers

## Typography Strategy

Typography should become a more important part of the visual identity.

### Direction

- large English headline in display style
- Chinese body and explanation content kept readable
- stronger contrast between display, section title, metadata, and body text
- more intentional spacing and line lengths

The typography should help create a brand feel even before motion and decoration are noticed.

## Color Strategy

The current palette is too singular and too soft. The redesign should use a wider but controlled palette.

### Light palette direction

- near-white background
- cool slate text
- pale blue or cyan glow accents
- neutral cards with soft border contrast

### Dark palette direction

- deep blue-black background
- cool off-white text
- cyan / electric blue highlight tones
- restrained glow on interactive and featured elements

Accent colors should be used strategically, not everywhere.

## Technical Identity Cues

To make the site feel more aligned with a programmer’s taste, the redesign should include subtle technical cues such as:

- measured metadata treatment
- terminal or interface-inspired separators
- system panel framing
- structured labels
- clear content hierarchy
- stronger project and writing categorization

These cues should support the identity without turning the site into a novelty hacker aesthetic.

## Blog Consistency

The blog added in the previous phase should remain and visually align with the redesign.

Blog pages should inherit:

- the theme system
- the stronger typography hierarchy
- the more editorial technical styling
- improved metadata presentation

The blog should feel like part of the same product, not a fallback template.

## Responsiveness

The redesign must still work well on:

- desktop
- tablet
- mobile

### Responsive priorities

- hero should reflow cleanly without losing impact
- technical decoration should not break mobile readability
- cards should stack with retained hierarchy
- theme toggle should remain accessible

## Constraints

The redesign must remain:

- static-export compatible
- maintainable for the user
- performance-conscious
- easy to deploy with the current workflow

It should not require:

- backend services
- runtime animation libraries unless clearly justified
- complex CMS integration

## Final Recommendation

Proceed with a full visual redesign that upgrades the site from a minimal personal portfolio into a more dynamic technical brand homepage with dual themes, stronger hierarchy, medium-strength motion, and a more developer-aligned identity.
